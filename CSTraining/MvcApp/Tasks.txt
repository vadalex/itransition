Задания выполняются по шагам. Каждый следующий шаг является продолжением и доработкой того что было сделано на предыдущем шаге.
	1)Сделать в VS 2012(2013) два проекта (Web Forms ASP.Net и MVC Asp.Net). Оба должны делать одно и тоже: 
		База данных MS SQL, в ней создать несколько таблиц со связями, например: Продукты, Производители продуктов, Покупатели, Покупки (много ко многим продукты к покупателям). В базе должны быть проставлены Foregnkeys, primary keys, id должно быть autoincrement)
		Для работы с данной базой данных использовать ORM -  Entity Framework. 
		Оба веб проекта должны выполнять простейшие CRUD операции для записей в таблице (create, read, update, delete) и делать это одинаково. Также должна быть простейшая валидация данных
	2)Вынести работу с базой данных в отдельный проект (DataAccess) для работы с базой данных использовать паттерн Repository. При этом базовая логика с базовой реализацией репозитория должна быть реализована в базовом классе с использованием generic (BaseRepository<T>). Оба веб проекта должны обращаться к базе данных только через новый проект.
	3)Сделать 3х уровневое приложение (DataAccess, Service, Web)
	4)Сделать интерфейсы для доступа к разным слоям приложения. Реализации интерфейсов в приложении получать через классы фабрики. Интерфейсы вынести в отдельный проект (Contracts)
	5)Заменить фабрики на IoC контейнер (Unity).
	6)Логирование и обработка ошибок в файл (NLog или Log4Net). Должна быть одна точка для логирования и обработки ошибок на всех слоях приложения. Exception должен логироваться вместе со всеми InnerException.
	7)АОП (Сделать подсчет вызвова каких-нибудь методов) (Enterprise blocks, Policy Injection)
	8)Сделать полнотекстовый поиск на Lucene.
	9)Сделать авторизацию Facebook.
	10)Сделать страницу для просмотра записей одной из таблиц, на которой список будет с Pagging и Search через Ajax (в MVC это JQuery, в Webforms - UpdatePanel).
